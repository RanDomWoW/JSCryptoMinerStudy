		<h1>Welcome to FreeDOS</h1>

<div style="float:right"><a href="http://www.freedos.org/images/screenshots/freedos12.png"><img alt="FreeDOS 1.2" src="http://www.freedos.org/images/screenshots/thumbs/freedos12.png" /></a></div>

<p>FreeDOS is a complete, free, DOS-compatible operating system that you can use to play classic DOS games, run legacy business software, or develop embedded systems. Any program that works on MS-DOS should also run on FreeDOS.</p>

<p>It doesn&rsquo;t cost anything to download and use FreeDOS. You can also share FreeDOS for others to enjoy! And you can view and edit our source code, because all FreeDOS programs are distributed under the GNU General Public License or a similar open source software license.</p>

<p><a href="http://wiki.freedos.org/">Learn more &raquo;</a></p>

<p><strong><a href="/download">Download FreeDOS 1.2 &raquo;</a></strong></p>

<h4><a href="https://sourceforge.net/p/freedos/news/2018/03/running-freedos-on-the-raspberry-pi-sort-of/">Running FreeDOS on the Raspberry Pi (sort of)</a></h4>
<time datetime="2018-03-13T18:38+0000">2018-03-13  6:38pm</time>
<div class="markdown_content"><p>A question I seem to get a lot, at least recently, is "Can I run FreeDOS on the Raspberry Pi?" The short answer is "no" because like any DOS, FreeDOS needs an Intel CPU and PC BIOS. The Raspberry Pi doesn't have either of these. The Raspberry Pi is based on an ARM CPU, which is a completely different architecture. But you can run FreeDOS on the Raspberry Pi if you use a PC emulator, like QEMU. FreeDOS runs fine in QEMU on Raspberry Pi, including games, although disk I/O is slow because the microSD card used in the Raspberry Pi isn't very fast. I've posted an article <a class="" href="http://freedos-project.blogspot.com/2018/01/how-to-run-freedos-on-raspberry-pi.html" rel="nofollow">How to run FreeDOS on Raspberry Pi</a> on the FreeDOS blog. And I've written a similar article for OpenSource about <a class="" href="https://opensource.com/article/18/3/can-you-run-dos-raspberry-pi" rel="nofollow">Running FreeDOS on the Raspberry Pi</a>.</p></div>
</article>
<article>
<h4><a href="https://sourceforge.net/p/freedos/news/2018/03/spotlight-as-easy-as-spreadsheet-for-dos/">Spotlight: As-Easy-As spreadsheet for DOS</a></h4>
<time datetime="2018-03-13T18:31+0000">2018-03-13  6:31pm</time>
<div class="markdown_content"><p>On the FreeDOS blog, I've written a <a class="" href="http://freedos-project.blogspot.com/2018/03/running-as-easy-as-on-freedos.html" rel="nofollow">"spotlight" article about As-Easy-As</a>, the shareware spreadsheet for DOS. As-Easy-As was my favorite DOS program of the 1980s and 1990s, no question. And I still have a great fondness for As-Easy-As, many years later. Whenever I install FreeDOS somewhere, I usually install As-Easy-As, as well. If you're new to FreeDOS and wonder, "What can I do with FreeDOS?" then I encourage you to try As-Easy-As. While the original version of As-Easy-As was shareware, TRIUS Software has since posted the original install files and <a class="" href="http://www.triusinc.com/forums/viewtopic.php?t=10" rel="nofollow">activation code for As-Easy-As 5.7</a>, including the full user manual, for free. Thanks, TRIUS! (This isn't "Free software" or even "open source software," but a "closed source" program that is now available at no cost.)</p></div>
</article>
<article>
<h4><a href="https://sourceforge.net/p/freedos/news/2018/02/updated-freecom-prerelease/">Updated FreeCOM prerelease</a></h4>
<time datetime="2018-02-24T00:01+0000">2018-02-24 12:01am</time>
<div class="markdown_content"><p>Bart Oldeman has been working to update the FreeDOS command.com shell (FreeCOM, our command-line interpreter) so it can be compiled with OpenWatcom, TurboC, and GNU gcc-ia16. Bart has updated the FreeCOM 0.84-pre4 prerelease with a few changes: * stabilized the ia16-elf-gcc version further * fixed "Out of memory" with the Open Watcom version * fixed build with Windows CMD (Tom Ehlert) * spelling fixes and Swedish translation updates (Anders Jonsson). You can find the <a class="" href="https://github.com/bartoldeman/freecom/releases/tag/com084pre4" rel="nofollow">new versions on GitHub</a>. Please test and let us know which one is most stable.</p></div>
</article>
<article>
<h4><a href="https://sourceforge.net/p/freedos/news/2018/02/dos-fork-of-fltk/">DOS fork of FLTK</a></h4>
<time datetime="2018-02-07T16:40+0000">2018-02-07  4:40pm</time>
<div class="markdown_content"><p>Mark Olesen recently announced his DOS fork of FLTK using Allegro4 and DJGPP.  "fltkal is a DOS fork of the Fast Light Toolkit (FLTK) using Allegro4 and DJGPP. The fork required renaming files to 8.3 conventions. Other than that, however, the API is compatible with the 1.3.x and 1.4.x branch."  You can see more at <a class="" href="https://github.com/markjolesen/fltkal" rel="nofollow">fltkal on Github</a>.<br/>
<em>This seems to be a separate effort than <a class="" href="https://sourceforge.net/projects/fltk-dos/">FLTK for DOS</a> at SourceForge.</em></p></div>
</article>
<article>
<h4><a href="https://sourceforge.net/p/freedos/news/2018/02/freecom-084-pre3-prerelease/">FreeCOM 0.84-pre3 prerelease</a></h4>
<time datetime="2018-02-04T22:56+0000">2018-02-04 10:56pm</time>
<div class="markdown_content"><p>Bart Oldeman writes: "As some of you know I spent some time fixing various bugs in FreeCOM. We've had the awkward situation of still having an old 2006 version in distributions but the newer versions had too many bugs (e.g. loadhigh, ren "myfile myfile.txt", strange dir output depending on the country setting). However the newer versions support LFNs much better. So for testing I uploaded a new prerelease <a class="" href="https://github.com/bartoldeman/freecom/releases/tag/com084pre3" rel="nofollow">on github</a>. There are three non-UPXed xms-swap command.com's in the binary zip, one compiled with Turbo C++ 1.01 (as the older version were), one cross-compiled with OW 1.9 and one cross-compiled with ia16-elf-gcc. Please test and let me know how stable they are in your testing."</p>
<p>Bart adds: "As a bonus for the XMS swap version, the resident footprint has been reduced from 3008 bytes (old Turbo C version) to 2304 bytes (new OW version)."</p></div>
