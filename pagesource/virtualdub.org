utility for 32-bit and 64-bit Windows platforms (98/ME/NT4/2000/XP/Vista/7), licensed under the <a href="gpl.html">GNU General Public License (GPL)</a>.&nbsp; It lacks the editing power of a general-purpose editor such as Adobe Premiere, but is streamlined for fast linear operations over video.&nbsp; It has batch-processing capabilities for processing large numbers of files and can be extended with <a href="virtualdub_filters.html">third-party video filters</a>.&nbsp; VirtualDub is mainly geared toward processing AVI files, although it can read (not write) MPEG-1 and also handle sets of BMP images.</p> <p> I basically <a href="virtualdub_history.html">started VirtualDub in college</a> to do some quick capture-and-encoding that I wanted done; from there it's basically grown <p>MSDN has code to detect whether a controller device enumerated by the DirectInput API is also an XInput device, so that programs can avoid reporting the same controller twice when both APIs are supported:</p><p><a href="http://msdn.microsoft.com/en-us/library/windows/desktop/ee417014%28v=vs.85%29.aspx">http://msdn.microsoft.com/en-us/library/windows/desktop/ee417014%28v=vs.85%29.aspx</a></p><p>Be warned, there is a memory leak in the supplied IsXInputDevice() function: the VARIANT returned by IWbemClassObject::Get() needs to be freed with VariantClear() after the device string is checked. This is hard to notice if you just drop in the code, as it only triggers on device enumeration and that likely only happens on startup or when a device change notification is received. I only noticed it because I temporarily modified my device enumeration code to run in a loop to see how fast (or slow) the WMI-based check would be.</p> <a href="/blog/pivot/entry.php?id=386#body">(Read more....)</a>
