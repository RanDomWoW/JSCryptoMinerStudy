	<dt id="news-17334678">2017-09-15: The first 700k: dblp starts integrating ORCIDs</dt>
	<dd class="expandable"><div class="expand-head"><p>ORCID (<a href="https://orcid.org/">https://orcid.org</a> is a widely used persistent identifier scheme for researchers. As of September 2017, there are more than 3.8 million registered ORCIDs of which about 800,000 have at least one publication listed in the public ORCID corpus. Many publishers will now ask their authors to provide an ORCID when submitting or publishing a paper. We encourage all researchers to <a href="https://orcid.org/register">register and map their ORCID with their publications</a>.</p></div><div class="expand-body"><p>A few weeks ago, we started first experiments with integrating ORCIDs into dblp. At the moment, there are about 700,000 signatures (i.e., author-publication pairs) in dblp for which we know an ORCID. This is about 6% of all signatures. We expected ORCIDs to help us identify authors and create clear, unambiguous author bibliographies (see <a href="http://dblp.org/faq/1474783">How does dblp handle homonyms and synonyms</a>). We were not disappointed. For the initial import, we found in our corpus</p><ul><li>600 cases where an author profile was related to more than one ORCID. This indicates that the profile actually lists publications from different authors.</li><li>5000 cases where the same ORCID appears in more than one author profile.</li></ul><p>We are currently processing these cases. While we found several cases where ORCID information is wrong (e.g., authors accidentally claiming publications that where written by someone else), the data is very reliable in general. However, according to our philosophy, manual confirmation is needed. This will take some time.</p><p>A detailed description on ORCIDs on the dblp web interface and in the data dump can be found at <a href="http://dblp.org/faq/17334571">in our F.A.Q.s</a>. </p><p>We will keep you posted on development. </p></div></dd>
	<dt id="news-17334612">2017-09-05: Further modifications to dblp's XML format</dt>
	<dd class="expandable"><div class="expand-head"><p>With the latest release of the dblp XML data dump, we made some extensions to the document type definition (DTD) of the XML format. The goal of these changes is to lay the groundwork for a number of planned extensions of the dblp data model that will be realized at a later date.</p></div><div class="expand-body"><p>These changes will only affect you if you use:</p><ul><li>the daily XML data dump, available at <a class="external-link" href="http://dblp.org/xml/" rel="nofollow">http://dblp.org/xml/</a>,</li><li>any future snapshot releases of the XML dump, available at <a class="external-link" href="http://dblp.org/xml/release/" rel="nofollow">http://dblp.org/xml/release/,</a> or</li><li>the live XML web API, e.g. <a class="external-link" href="http://dblp.org/rec/xml/journals/pvldb/Ley09.xml" rel="nofollow">http://dblp.org/rec/xml/journals/pvldb/Ley09.xml</a></li></ul><p>Regular users of the web site or other export formats &#8211; like the BibTeX export &#8211; are not affected.</p><p>As always, the latest version of the DTD can be found at:</p><ul><li><a class="external-link" href="http://dblp.dagstuhl.de/xml/dblp.dtd" rel="nofollow">http://dblp.org/xml/dblp.dtd</a></li></ul><p>All upcoming as well as earlier versions of the DTD file will be preserved along with the persistent snapshot releases at:</p><ul><li><a class="external-link" href="http://dblp.org/xml/release/" rel="nofollow">http://dblp.org/xml/release/</a></li></ul><p>A brief summary of the recent changes is available at:</p><ul><li><a class="external-link" href="http://dblp.org/xml/CHANGES.txt" rel="nofollow">http://dblp.org/xml/CHANGES.txt</a></li></ul><p>Detailed Information on the newly added attributes can be found in the F.A.Q.:</p><ul><li><p><a href="https://dblp.org/faq/16154937">What do I find in dblp.xml</a></p></li><li><p><a href="https://dblp.org/faq/17334559">How are data annotations used in dblp.xml</a></p></li></ul></div></dd>

	<dt id="news-15401051">2016-08-31: Full-text search dblp, courtesy of Semantic Scholar</dt>
	<dd class="expandable"><div class="expand-head"><p>Thanks to the search API provided by our friends at the <a href="http://allenai.org/">Allen Institute for Artificial Intelligence (AI2)</a>, we have added a new full-text search option to dblp. By choosing "Semantic Scholar search" as your default search action in the drop down menu next to the search bar, searching will now retrieve documents from dblp based on matches within the full texts as indexed by the <a href="https://www.semanticscholar.org">Semantic Scholar</a> service. Since dblp usually does not have access to the full texts of research articles, this is something that has not been possible before. <a href="https://dblp.org/search/semscholar?q=full-text+search">Please try it for yourself.</a> </p></div><div class="expand-body"><p>The new search option is currently an experimental feature (which may still break once in a while) and we will be happy to hear your thoughts and comments about it.</p></div></dd>
