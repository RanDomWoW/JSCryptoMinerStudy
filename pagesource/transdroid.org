		<h2 class="entry-title"><a href="https://www.transdroid.org/2015/06/25/material-is-here/" rel="bookmark">Material is here</a></h2>	</header><!-- .entry-header -->

	<div class="entry-content">
		<p><img class="alignright wp-image-737" src="http://www.transdroid.org/wp-content/uploads/2015/06/transdroid-2.5.1-main-576x1024.png" alt="Transdroid 2.5.1" width="281" height="500" srcset="https://www.transdroid.org/wp-content/uploads/2015/06/transdroid-2.5.1-main-576x1024.png 576w, https://www.transdroid.org/wp-content/uploads/2015/06/transdroid-2.5.1-main-169x300.png 169w, https://www.transdroid.org/wp-content/uploads/2015/06/transdroid-2.5.1-main.png 1080w" sizes="(max-width: 281px) 100vw, 281px" />After a long time of some mere bugfixes, following by a long time of slow work on the promised refresh&#8230; Transdroid 2.5 is finally here!</p>
<p>The obvious change in the new Material design. Both light and dark themes are now styled in the design language that was introduced in Android Lollipop. While more is to come, the app should feel a little more modern in both graphical appeal and feel. As always, feel free <a href="https://github.com/erickok/transdroid">Transdroid on GitHub</a>!</p>
<p>Under the hood I have solved an array of bugs and incompatibilities. User of rTorrent, qBittorrent or Aria2? The new versions should be compatible with the latest and greatest of your favourite torrent client. Translations have been updated too. The Search Module was also improved. Several private and public trackers have been added for your direct search pleasure.</p>
<p><img class="alignright wp-image-417" src="http://www.transdroid.org/wp-content/uploads/2009/04/transdroid-screenshot-0.13.1-main-200x300.png" alt="Transdroid 0.13 from 2009" width="167" height="250" srcset="https://www.transdroid.org/wp-content/uploads/2009/04/transdroid-screenshot-0.13.1-main-200x300.png 200w, https://www.transdroid.org/wp-content/uploads/2009/04/transdroid-screenshot-0.13.1-main.png 320w" sizes="(max-width: 167px) 100vw, 167px" />Finally, for now, I am updating the website and documentation. Some of it is from 2009 still. And I think you&#8217;d agree that SOME things have changed since then&#8230;</p>

	<header class="entry-header">
		<h2 class="entry-title"><a href="https://www.transdroid.org/2014/03/14/some-private-tracker-love/" rel="bookmark">Some private tracker love</a></h2>	</header><!-- .entry-header -->

	<div class="entry-content">
		<p>I just released Transdroid 2.1.3. It includes bug fixes, such as for the rTorrent file upload restriction, but more importantly it supports the new Torrent Search 2.0. With the 2.0 release of the Search module I introduced support for private sites to function just as the public sites that were already supported nicely, fully integrating in the search feature of Transdroid and other apps.</p>
<p>When you open the new Torrent Search interface you can enable and disable public sites in case you are not interested in their content, and Transdroid will not show them in the search results. More importantly, you can provide the credentials to private trackers that you have an existing account for to get the same smooth search experience in Transdroid for your private sites.</p>
<p>Since every private tracker will have a different code and login method, support for each of them has to be custom build. Currently I only support IPTorrents and Bit-HDTV. If you have a private site that you like support for, you may either implement it yourself of course if you know some Java (you can use <a href="https://github.com/erickok/transdroid-search/blob/master/src/org/transdroid/search/IpTorrents/IpTorrentsAdapter.java">the IPTorrents class</a> to start) and send me a pull request <a href="https://github.com/erickok/transdroid-search">on GitHub</a>. It is quite simple really. No coding skills (or lazy)? If you provide me with an account I might add it myself &#8212; although my time is very limited so I can give no guarantees.</p>
	</div><!-- .entry-content -->

	<header class="entry-header">
		<h2 class="entry-title"><a href="https://www.transdroid.org/2013/11/18/big-update-lots-of-fixes/" rel="bookmark">Big update, lots of fixes</a></h2>	</header><!-- .entry-header -->

	<div class="entry-content">
		<p><strong>UDATE</strong>: Transdroid does not support rTorrent with Digest authentication in this 2.1.0 release. I&#8217;ve fixed this for the next release, hopefully out at the end of this week!</p>
<p>Big update today to Transdroid and its little Play Store-cousin Transdrone. Most importantly it fixes MANY annoyances in both the UI and connectivity.</p>
<p>Using uTorrent, rTorrent, Transmission, Deluge, Bitflu, qBittorrent, Torrentflux-b4rt? Quite likely&#8230; and the new version should be quite a bit more stable for you. Moreover, Deluge has not support for labels (viewing and setting) and uTorrent and Deluge can force data recheck.</p>
<p>The UI has seen many little fixes like selection improvements, search input box that properly hides and much more. Remember the option to treat non-trasferring torrents as inactive in Transdroid 1? This is back: enable the &#8216;Treat dormant torrents as inactive&#8217; option in System settings and every torrent that has no data transfer (0KB/s) going on will be listed as inactive. Especially useful if you have many seeding torrents that are not connected to any leecher. Finally, the widget now contains buttons for pausing or resuming all torrents on the server and can optionally show the server status statistics, that is, the number of and speed of your active uploads and download.</p>
<p>Perhaps your favorite feature is not here or you still have a poor connection? Add your issue to <a title="GitHub issue tracker" href="https://github.com/erickok/transdroid">the GitHub issue tracker</a>. Thanks!</p>
	</div><!-- .entry-content -->

	<header class="entry-header">
		<h2 class="entry-title"><a href="https://www.transdroid.org/2013/10/17/trans-what/" rel="bookmark">Trans&#8230;what?</a></h2>	</header><!-- .entry-header -->

	<div class="entry-content">
		<p>Today marks a big day in the history of Transdroid. For the existing users, some of you who use Transdroid many, many years already, will receive an update notification for Transdroid 2, the first non-beta release. The whole UI has received an overhaul in true modern &#8216;Holo&#8217; style fashion. Not only does this mean a more attractive and hopefully nicer to use app, but it also solved many interface bugs and makes the app ready for the future. This includes a modern widget, multi-select of torrents and files and much more. UPDATE: You can use the settings export and import feature to transfer your server, RSS and search settings.</p>
<p>But before we look into the future, a short trip to the past. Two years ago Transdroid was removed from the Play Store &#8212; then still called Android Market &#8212; and I am very happy to announce that today you can download Trandroid again directly from the official Android Play Store. Transdroid? Well, not exactly. To prevent potential problems I have decided that the Play Store version will not included torrent search and RSS feeds. To signify this clear difference I&#8217;ve given it a new name. Meet Transdroid&#8217;s little brother: <a title="Transdrone on the Play Store" href="https://play.google.com/store/apps/details?id=org.transdroid.lite">Transdrone</a>. Existing users who can live without integrated search, but more importantly new users who never heard of Transdroid, can now get a truly capable torrent manager right from the Play Store. And for those who love the search: the full Transdroid will always remain available <a title="Download Transdroid" href="http://www.transdroid.org/download/">right here</a>. You&#8217;ll get update notifications right from the app.</p>
<p><a href="https://play.google.com/store/apps/details?id=org.transdroid.lite"><img src="https://developer.android.com/images/brand/en_generic_rgb_wo_60.png" alt="Get it on Google Play" /></a></p>
<p>Where from here? Well, development will not stop, only calm down a little. There are some bits and pieces that Transdroid 1 has and the new version doesn&#8217;t &#8212; most of them will be added in due course. New stuff will appear too. Got some idea for yourself? Share it on <a title="Issue tracker on GitHub" href="https://github.com/erickok/transdroid/issues">the new issue tracker</a>.</p>
<p>Finally: if you upgrade you might encounter some small issues in certain special cases. Don&#8217;t hesitate to <a title="Mail transdroid@2312.nl" href="mailto:transdroid@2312.nl">contact me via mail</a> or <a title="Support forum on transdroid.org" href="http://www.transdroid.org/support-forum/">on the forums</a>. Found a real bug/crash? Use the error report feature (now found in the system settings) or <a title="Issue tracker on GitHub" href="https://github.com/erickok/transdroid/issues">leave a log on GitHub</a>.</p>
	</div><!-- .entry-content -->
