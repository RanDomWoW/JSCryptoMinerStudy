<a name="About"></a>
<h2>About</h2>

<p>The linux-based open-source mobile operating system Android is not only the most popular mobile operating system in the world,
it&rsquo;s also on the way to becoming a proprietary operating system. How is that?</p>

<p>While the core operating system is still released as part of the Android Open Source Project, the majority of core apps are not.
It gets worse: More and more libraries and APIs are only available on phones that run various Google apps pre-installed,
effectively locking third-party apps to the Google ecosystem. For these reasons Android is <a href="http://arstechnica.com/gadgets/2013/10/googles-iron-grip-on-android-controlling-open-source-by-any-means-necessary/4/">described as being</a> a &ldquo;<em>look but don&rsquo;t touch</em>&rdquo; kind of open.</p>

<p>At this point, several popular open-source applications already require some of Google&rsquo;s proprietary libraries to be installed.
Increasing demand in the free software community in addition to severe problems in Google&rsquo;s proprietary software discovered by the Android modding community, have led to the development of a free software clone of Google&rsquo;s proprietary core libraries and applications - the microG Project was born.</p>

<p>Although most microG components are far from complete, users are amazed by the results. Free software users got extended application support,
privacy-caring users can reduce or monitor data that is sent to Google and especially older phones can expect some battery life improvements.
microG is not only used on real devices, but also replaces Google tools in test emulators and is even used in virtual mobile infrastructure.</p>

<a name="Components"></a>
<h2>Components</h2>

<ul>
<li><p><strong>Service Core (GmsCore)</strong> is a library app, providing the functionality required to run apps that use Google Play Services or Google Maps Android API (v2).</p>

<p><a href="https://github.com/microg/android_packages_apps_GmsCore/wiki">More details and installation instructions</a></p></li>
<li><p><strong>Services Framework Proxy (GsfProxy)</strong> is a small helper utility to allow apps developed for Google Cloud to Device Messaging (C2DM) to use the compatible Google Cloud Messaging service included with GmsCore.</p>

<p><a href="https://github.com/microg/android_packages_apps_GmsCore/wiki">Read GmsCore documentation for details</a></p></li>
<li><p><strong>Unified Network Location Provider (UnifiedNlp)</strong> is a library that provides Wi-Fi- and Cell-tower-based geolocation to applications that use Google&rsquo;s network location provider.
It is included in GmsCore but can also run independently on most Android systems.</p>

<p><a href="https://github.com/microg/android_packages_apps_UnifiedNlp/blob/master/README.md">More details and installation instructions</a></p></li>
<li><p><strong>Maps API (mapsv1)</strong> is a system library, providing the same functionality as now deprecated Google Maps API (v1).</p>

<p><a href="https://github.com/microg/android_frameworks_mapsv1">More details and installation instructions</a></p></li>
<li><strong>Store (Phonesky)</strong> is a frontend application providing access to the Google Play Store to download and update applications. Development is in early stage and there is no usable application yet.</li>
</ul>
