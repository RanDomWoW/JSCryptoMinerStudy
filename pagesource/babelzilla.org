			<td valign="top" colspan="2">
				<div align="justify" style="color: #000000"><strong style="font-family: Arial,Helvetica,sans-serif; font-size: 14px">Our site and community are dedicated to provide <a href="http://en.wiktionary.org/wiki/localization#Noun" target="_blank">localizations</a> to extensions (for Mozilla-family apps using Gecko) </strong></div>
<div align="justify" style="color: #000000">
<ul>
    <li><strong style="font-family: Arial,Helvetica,sans-serif; font-size: 12px">developers  (or their representative)  upload their extension(s) using the <a href="index.php?option=com_wts&amp;Itemid=88">Web Translation System (WTS)</a></strong></li>
    <li><strong style="font-family: Arial,Helvetica,sans-serif; font-size: 12px">then volunteer translators around the world provide localization for free.</strong><strong class="main"> That simple.</strong></li>
</ul>
<br /> <br /> <br /> <br /> <strong>For all nations which don't have English as national language it is very  important to have their locale included in the official author's build,  because if a user looks for an extension in big web sites such as <a href="https://addons.mozilla.org">addons.mozilla.org</a></strong> <span style="font-weight: bold;">they will find the extension in their language immediately. Moreover, if  the locale is not included in the official build, the user will find XUL  errors if they automatically update the extension with Firefox or  another Mozilla application.</span><br />
<div align="justify"><strong>  Because of this fact, we decided to start a world project, named <a href="http://babelzilla.org/" target="_blank">BabelZilla</a>, with the goal to internationalize the extensions!<br />  Thus we contribute to spread open source applications worldwide.<br /></strong></div>
