						<h1 class="entry-title">
				<a href="https://nagios-plugins.org/nagios-plugins-2-2-1-released/" rel="bookmark">Nagios Plugins 2.2.1 Released</a>
			</h1>
								</header><!-- .entry-header -->

				<div class="entry-content">
			<p>The <strong>Nagios Plugins Development Team</strong> is proud to announce that <a href="https://nagios-plugins.org/download/nagios-plugins-2.2.1.tar.gz">nagios-plugins 2.2.1</a> has been released and is available for <a href="https://nagios-plugins.org/download/nagios-plugins-2.2.1.tar.gz">download</a>.</p>
<p><u><strong>Fixes:</strong></u></p>
<ul>
<li>check_users: not accepting zero as the threshold</li>
<li>check_http: reports warning where it should report ok with -e</li>
<li>check_snmp: does not work with -6 &#8211;ipv6 flags</li>
<li>check_swap: threshold calculation in bytes requires subtracting 65</li>
<li>check_uptime: fixed backward help text for thresholds</li>
<li>check_http: Don’t prematurely report success when checking HTTP TLS cert validity</li>
<li>check_http: fix parsing the last header</li>
<li>check_mailq: Fix for Postfix and better Sudo Checking</li>
<li>configure.ac: Fix spelling error</li>
<li>check_ntp_peer: requires newline when there is a socket timeout (fix in netutils.c)</li>
<li>check_users: segmentation fault if both thresholds are not provided</li>
<li>check_dns: DNS CRITICAL &#8211; expected ‘{hostname}.’ but got &#8216;name = {hostname}.&#8217;</li>
<li>check_mailq: Nullmailer Regex is not working for Ubuntu 16.04</li>
<li>check_swap: Downstream Fedora patch: Prevent check_swap from returning OK, if no swap activated</li>
<li>Building RPMs on Amazon Linux &#8211; Add &#8216;install-root&#8217; on line 165 of spec file</li>
<li>check_http: Memory allocation error</li>
</ul>
<p><a href="https://nagios-plugins.org/download/nagios-plugins-2.2.1.tar.gz">Download Nagios-Plugins 2.2.1 here.</a></p>
<p>Thanks to all who have contributed toward this release. If you need assistance installing or using the plugins, please visit our general support forum. Bug reports, feature requests, and additional comments are welcome and may be posted to our plugin development forum.</p>

			</h1>
								</header><!-- .entry-header -->

				<div class="entry-content">
			<p>The <strong>Nagios Plugins Development Team</strong> is proud to announce that <a href="https://nagios-plugins.org/download/nagios-plugins-2.2.0.tar.gz">nagios-plugins 2.2.0</a> has been released and is available for <a href="https://nagios-plugins.org/download/nagios-plugins-2.2.0.tar.gz">download</a>.</p>
<p><u><strong>Enhancements:</strong></u></p>
<ul>
<li>check_flexlm: if <code>-F &lt;license file&gt;</code> is not specified, will use <code>LM_LICENSE_FILE</code> environment var</li>
<li>check_load: Added per cpu load average message</li>
<li>check_smtp: add -L flag to support LMTP (LHLO instead of HELO/EHLO)</li>
</ul>
<p><u><strong>Fixes:</strong></u></p>
<ul>
<li>check_http: -e breaks -f</li>
<li>check_mrtg: Add state to status output</li>
<li>check_ping: ping runs 30 times when host is down</li>
<li>check_icmp: does not have the -p argument in the help</li>
<li>check_dns: Segfaulting with timeout &gt; 26 sec</li>
<li>check_disk: missing -lrt on Solaris</li>
<li>check_http: segmentation fault</li>
<li>check_http: help text update for virtual hosts</li>
<li>check_snmp: Thresholds were being shown twice</li>
<li>check_hpjd: some jd 610 cards have a false flag that printer is offline</li>
<li>check_http: Handle reference redirect like //www.site.org/test</li>
<li>check_disk: alerts issued too soon</li>
<li>fix: Allocator sizeof operand mismatch</li>
<li>fix: Dead assignment</li>
<li>Shellcheck: fix most of the shellcheck warnings.</li>
<li>check_ntp: touch ntp servers at most once every seconds</li>
<li>check_dns: authoritative test (-A) is broken</li>
<li>check_dns: reports TXT records incorrectly</li>
<li>check_file_age: does not handle filenames WITHOUT space!</li>
<li>de,fr.po: fix syntax errors end-of-line within string</li>
<li>lib/parse_ini.c: fix gcc warning: implicit declaration of function ‘idpriv_temp_drop’ and &#8216;idpriv_temp_restore’</li>
<li>add openssl 1.1 support</li>
</ul>
<p><a href="https://nagios-plugins.org/download/nagios-plugins-2.2.0.tar.gz">Download Nagios-Plugins 2.2.0 here.</a></p>
<p>Thanks to all who have contributed toward this release. If you need assistance installing or using the plugins, please visit our general support forum. Bug reports, feature requests, and additional comments are welcome and may be posted to our plugin development forum.</p>

						<h1 class="entry-title">
				<a href="https://nagios-plugins.org/nagios-plugins-2-1-2-released/" rel="bookmark">Nagios Plugins 2.1.2 Released</a>
			</h1>
								</header><!-- .entry-header -->

				<div class="entry-content">
			<p>The <strong>Nagios Plugins Development Team</strong> is proud to announce that <strong><a href="https://nagios-plugins.org/download/nagios-plugins-2.1.2.tar.gz">nagios-plugins 2.1.2</a></strong> has been released and is available for <strong><a href="https://nagios-plugins.org/download/nagios-plugins-2.1.2.tar.gz">download.</a></strong></p>
<p><u><strong>Enhancements:</strong><br />
</u>&#8211; check_snmp&#8217;s performance data now also includes warning/critical thresholds<br />
&#8211; New check_snmp &#8220;-N&#8221; option to specify SNMPv3 context name<br />
&#8211; New check_nt &#8220;-l&#8221; parameters: seconds|minutes|hours|days<br />
&#8211; New check_mailq -s option which tells the plugin to use sudo<br />
&#8211; New -W/-C option for check_ldap to check number of entries<br />
&#8211; The check_http -S/&#8211;ssl option now accepts the arguments &#8220;1.1&#8221; and &#8220;1.2&#8221; to force TLSv1.1 and TLSv1.2 connections, respectively<br />
&#8211; The check_http -S/&#8211;ssl option now allows for specifying the desired protocol with a &#8220;+&#8221; suffix to also accept newer versions<br />
&#8211; New check_disk &#8220;-v&#8221; option to show troubled partition in verbose mode<br />
&#8211; Added a parameter -w (&#8211;max_warning) to check_log.sh defining upper value to return a warning code<br />
&#8211; Add support for LDAP URIs in check_ldap<br />
&#8211; Provide performance data in check_file_age<br />
&#8211; Added &#8211;hostname support to check_by_ssh<br />
&#8211; Add check_ifstatus option to ignore interfaces by name<br />
&#8211; Introduce support for SNMPv3 context using &#8220;-N&#8221; option in check_snmp:<br />
&#8211; Added IPv6 support to check_snmp.c<br />
&#8211; Added support for checking SSL-Websites through Proxies to check_http</p>
<p><u><strong>Fixes:</strong></u><br />
&#8211; check_dig can now also use &#8220;drill&#8221; instead of &#8220;dig&#8221;<br />
&#8211; check_dig now honors the -4 and -6 switches<br />
&#8211; Fix check_jabber to work with Openfire servers<br />

<p class="" style="padding-left: 30px;">-t &lt;interval&gt;:&lt;state&gt;</p>
<p class="" style="padding-left: 30px;">Interval is a positive integer.<br />

-t 5:0 = 5 seconds timeout with state ok<br />

			<p>The Nagios Plugins Development Team is proud to announce that <a title="download nagios-plugins 2.0.1" href="http://nagios-plugins.org/downloads/?utm_source=nagios-plugins.org&amp;utm_medium=post&amp;utm_content=Nagios%20Plugins%202.0.1%20Released&amp;utm_campaign=Nagios%20Plugins" target="_blank">nagios-plugins 2.0.1</a> has been released and is available for <a title="download nagios-plugins 2.0.1" href="http://nagios-plugins.org/downloads/?utm_source=nagios-plugins.org&amp;utm_medium=post&amp;utm_content=Nagios%20Plugins%202.0.1%20Released&amp;utm_campaign=Nagios%20Plugins" target="_blank">download</a>. This is mostly a maintenance release, focusing on check_dns stability and some bugs in the new check_uptime plugin. A full list of included enhancements and fixes are listed below:</p>
<p><span style="text-decoration: underline;"><strong>ENHANCEMENTS</strong></span></p>
<ul>
<li>check_snmp.c &#8211; Added thresholds to performance data (jccomputing)</li>
<li>check_http.c &#8211; Added */* MIME support (Alex Gottschalk)</li>
<li>check_mailq.pl &#8211; Added autodetection for mailq implementation (evgeni)</li>
<li>check_mailq.pl &#8211; Added autodetected version to output (evgeni)</li>
</ul>
<p><span style="text-decoration: underline;"><strong>FIXES</strong></span></p>
<ul>
<li>check_dns.c &#8211; Record type corrections for default check (sreinhardt)</li>
<li>utils.c &#8211; Changed comment type &#8211; now supported by AIX and should no longer fail compile (hkpatv)</li>
<li>check_uptime.c &#8211; Fixed timeout option (abrist)</li>
<li>check_uptime.c &#8211; Fixed help output and updated verbose usage (abrist)</li>
<li>check_uptime.c &#8211; Removed executable permissions on check_uptime.c (abrist)</li>
<li>check_uptime.c &#8211; Removed old code (madrisian)</li>
<li>check_pgsql.c &#8211; Fixed help output (weiss)</li>
<li>lib/Makefile.am &#8211; Avoid deprecated includes (madrasian)</li>
<li>lib/tests/Makefile.am &#8211; Avoid deprecated includes (madrasian)</li>
<li>plugins/Makefile.am &#8211; Avoid deprecated includes (madrasian)</li>
<li>check_apt.c &#8211; Removed the verbose flag suggestion (skottler)</li>
<li>check_oracle.c &#8211; Changed regex to compensate for tns string match edge cases (waja)</li>
<li>check_snmp.c &#8211; Always privides the security name when noAuthPriv is specified (catharsis)</li>
</ul>
<p>Thanks to all who have contributed toward this release. If you need assistance installing or using the plugins, please visit our <a title="visit general support forum" href="http://support.nagios.com/forum/viewforum.php?f=7?utm_source=nagios-plugins.org&amp;utm_medium=post&amp;utm_content=Nagios%20Plugins%202.0.1%20Released&amp;utm_campaign=Nagios%20Plugins" target="_blank">general support forum</a>. Bug reports, feature requests, and additional comments are welcome and may be posted to our <a title="visit plugin development forum" href="http://support.nagios.com/forum/viewforum.php?f=35?utm_source=nagios-plugins.org&amp;utm_medium=post&amp;utm_content=Nagios%20Plugins%202.0.1%20Released&amp;utm_campaign=Nagios%20Plugins" target="_blank">plugin development forum</a>.</p>
