<div class="section" id="what-is-happening">
<h2>What is happening<a class="headerlink" href="#what-is-happening" title="Permalink to this headline">¶</a></h2>
<div class="section" id="varnish-6-0-0-is-released">
<h3>2018-03-15 - Varnish 6.0.0 is released<a class="headerlink" href="#varnish-6-0-0-is-released" title="Permalink to this headline">¶</a></h3>
<p>Come and get it... <a class="reference internal" href="releases/rel6.0.0.html#rel6-0-0"><span class="std std-ref">Varnish Cache 6.0.0</span></a></p>
<p>Now with support for Unix Domain Sockets, better HTTP/2 support,
many bugfixes and ... Well read more about it all
<a class="reference external" href="/docs/6.0/whats-new/changes-6.0.html">here</a></p>
</div>
<div class="section" id="security-advisory-unlikely-data-leak">
<h3>2017-11-15 - Security Advisory: (Unlikely) data leak<a class="headerlink" href="#security-advisory-unlikely-data-leak" title="Permalink to this headline">¶</a></h3>
<p>Certain uncommon configurations of Varnish may leak data in
synthetic responses from <cite>vcl_backend_error{}</cite></p>
<p>Please see <a class="reference internal" href="security/VSV00002.html#vsv00002"><span class="std std-ref">VSV00002 Data leak - &#8216;-sfile&#8217; Stevedore transient objects</span></a></p>
<p>We have released Varnish <a class="reference internal" href="releases/rel4.1.9.html#rel4-1-9"><span class="std std-ref">Varnish Cache 4.1.9</span></a> and <a class="reference internal" href="releases/rel5.2.1.html#rel5-2-1"><span class="std std-ref">Varnish Cache 5.2.1</span></a> to fix this issue.</p>
</div>
<div class="section" id="varnish-5-2-0-is-released">
<h3>2017-09-15 - Varnish 5.2.0 is released<a class="headerlink" href="#varnish-5-2-0-is-released" title="Permalink to this headline">¶</a></h3>
<p>Come and get it... <a class="reference internal" href="releases/rel5.2.0.html#rel5-2-0"><span class="std std-ref">Varnish Cache 5.2.0</span></a></p>
</div>
<div class="section" id="security-advisory-denial-of-service">
<h3>2017-08-02 - Security Advisory: Denial of Service<a class="headerlink" href="#security-advisory-denial-of-service" title="Permalink to this headline">¶</a></h3>
<p>All varnish versions from 4.0.1 are vulnerable to a Denial of Service
attack, please see <a class="reference internal" href="security/VSV00001.html#vsv00001"><span class="std std-ref">VSV00001 DoS vulnerability</span></a></p>
</div>
<div class="section" id="security-releases-4-0-5-4-1-8-and-5-1-3">
<h3>2017-08-02 - Security releases: 4.0.5, 4.1.8 and 5.1.3<a class="headerlink" href="#security-releases-4-0-5-4-1-8-and-5-1-3" title="Permalink to this headline">¶</a></h3>
<p>Varnish versions 4.0.5, 4.1.8 and 5.1.3 are now available. See
<a class="reference internal" href="security/VSV00001.html#vsv00001"><span class="std std-ref">VSV00001 DoS vulnerability</span></a> for details.</p>
</div>
</div>
<div class="section" id="package-repository-status">
<h2>Package repository status<a class="headerlink" href="#package-repository-status" title="Permalink to this headline">¶</a></h2>
<p>The official Linux (apt/yum) package repositories are now located
at Packagecloud.io.
A list of all available repositories can be found at:
<a class="reference external" href="https://packagecloud.io/varnishcache">https://packagecloud.io/varnishcache</a></p>
<p>The old <a class="reference external" href="http://repo.varnish-cache.org/">http://repo.varnish-cache.org/</a> is going to be removed on the 31st of August.</p>
<p>For more details on packages, see <a class="reference internal" href="releases/index.html#releases"><span class="std std-ref">Releases &amp; Downloads</span></a></p>
</div>
<div class="section" id="privacy">
<h2>Privacy<a class="headerlink" href="#privacy" title="Permalink to this headline">¶</a></h2>
<p>You can access the varnish-cache homepages with HTTP or HTTPS as you
like.</p>
<p>We save the logfiles from our Varnish instance for a limited period,
in order to be able to debug problems.</p>
<p>The old homepage ran Google Analytics, and we have continued that here
for now, but unless we actually learn something useful from the data
they collect, it will be discontinued.</p>
