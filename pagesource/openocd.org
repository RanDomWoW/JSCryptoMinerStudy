				<h2><a href="http://openocd.org/2017/01/openocd-0-10-0-release-is-out/" rel="bookmark" title="Permanent Link to OpenOCD 0.10.0 release is out">OpenOCD 0.10.0 release is out</a></h2>
				<small>January 22nd, 2017 <!-- by Paul Fertser --></small>

				<div class="entry">
					<p>Hello,</p>
<p>I&#8217;m happy to announce the availability of <a href="https://sourceforge.net/projects/openocd/files/openocd/0.10.0/">OpenOCD version 0.10.0</a>, finally! For the important points regarding packaging please see <a href="https://sourceforge.net/p/openocd/mailman/message/35570895/">[1]</a>.</p>
<p>It was a long release cycle but it was also a fruitful one. The list of the most important changes follows.</p>
<p>OpenOCD is a community project and I want to take this opportunity to say <em>thank you</em> to all the contributors, regular and occassional, hobbyists and professionals, corporate-supported and those doing the work on their spare time. Please keep the flow of the patches going, both by sending new code and by <a href="http://openocd.zylin.com">reviewing</a> other&#8217;s contributions. It&#8217;s you who really decide what new features OpenOCD is going to get.</p>
<p>I also want to remind how important it is to communicate to the silicon vendors that you would like to see their devices supported upstream (or that you&#8217;re happy that they&#8217;re supported).</p>
<h4>Highlights of the changes made in the OpenOCD source archive release</h4>
<p>
JTAG Layer:</p>
<ul>
<li> New driver for J-Link adapters based on libjaylink (including support for FPGA configuration, SWO and EMUCOM)</li>
<li> FTDI improvements to work at 30MHz clock</li>
<li> BCM2835 native driver SWD and Raspberry Pi2 support</li>
<li> BCM2835 is set to 4ma drive, slow slew rate</li>
<li> ixo-usb-jtag (emulation of an Altera Bus Blaster I on Cypress FX2 IC) support</li>
<li> JTAG pass-through mode for CMSIS-DAP (including support for FPGA configuration)</li>
<li> OpenJTAG support for Cypress CY7C65215</li>
<li> connect_assert_srst support for SWD</li>
<li> Xilinx Virtex-II Series7 bitstream loading support</li>
<li> Use JEP106 data to decode IDs</li>
<li> Deprecated &#8220;ft2232&#8243; driver removed (use &#8220;ftdi&#8221; instead)</li>
<li> GPL-incompatible FTDI D2XX library support dropped (Presto, OpenJTAG and USB-Blaster I are using libftdi only now)</li>
<li> ZY1000 support dropped (unmaintained since long)</li>
<li> oocd_trace support dropped</li>
</ul>
<p>
Target Layer:</p>
<ul>
<li> ARMv7-A, Cortex-M, Cortex-A/R important fixes and improvements (allowing e.g. simultaneous debugging of A8 and M3 cores, JTAG WAIT support etc.)</li>
<li> ARM Cortex-A,R allow interrupt disable during single-step (maskisr command)</li>
<li> Semihosting support for ARMv7-A</li>
<li> ARM Cortex-M7 support</li>
<li> Intel Quark mcu D2000 support</li>
<li> Freescale LS102x SAP support</li>
<li> ThreadX RTOS support on ARM926E-JS</li>
<li> Cortex-M RTOS stack alignment fixes</li>
<li> FreeRTOS FPU support</li>
<li> uC/OS-III RTOS support</li>
<li> bridging semihosting to GDB&#8217;s File-I/O support</li>
<li> -defer-examine option added to target create command</li>
<li> verify_image_checksum command added</li>
</ul>
<p>
Flash Layer:</p>
<ul>
<li> Atmel SAM4S, SAM4N, SAM4C support</li>
<li> Atmel SAMV, SAMS, SAME (Cortex-M7) support</li>
<li> Atmel AT91SAMD handle reset run/halt in DSU, other fixes</li>
<li> Atmel AT91SAML21, SAML22, SAMC20/SAMC21, SAMD09 support</li>
<li> ST STM32F4x support</li>
<li> ST STM32F74x/76x/77x, STM32L4 support</li>
<li> ST STM32L0 categories 1, 2 and 5 support</li>
<li> Kinetis K02, K21, K22, K24, K26, K63, K64, K66 support</li>
<li> Kinetis KE, KVx, K8x families support</li>
<li> Kinetis FlexNVM handling</li>
<li> Kinetis flash protection, security, mass_erase improvements</li>
<li> Infineon XMC4xxx family support</li>
<li> Infineon XMC1000 flash driver</li>
<li> Energy Micro EFM32 Happy Gecko support</li>
<li> Energy Micro EFM32 debug interface lock support</li>
<li> Analog Devices ADuCM360 support</li>
<li> Unified Nuvoton NuMicro flash driver</li>
<li> NIIET K1921VK01T (Cortex-M4) support</li>
<li> Nordic Semiconductor nRF51 improvements</li>
<li> Spansion FM4 flash (including MB9BFx64/x65, S6E2DH) driver</li>
<li> Ambiq Micro Apollo flash driver</li>
<li> PIC32MX new device IDs, 17x/27x flash support</li>
<li> read_bank() and verify_bank() NOR flash internal API to allow reading (and verifying) non-memory-mapped devices</li>
<li> JTAGSPI driver to access SPI NOR flashes via a trivial FPGA proxy</li>
<li> Milandr read/verify for Info memory support</li>
<li> Various discrete SPI NOR flashes support</li>
<li> CFI 16-bit flash reversed endianness support</li>
</ul>
<p>
Board, Target, and Interface Configuration Scripts:</p>
<ul>
<li> Digilent JTAG-HS2, JTAG-HS3 interfaces configs</li>
<li> FTDI UM232H module as JTAG interface config</li>
<li> 100ask&#8217;s OpenJTAG interface config</li>
<li> MBFTDI interface config</li>
<li> XDS100v3 interface config</li>
<li> Freescale Vybrid VF6xx target config</li>
<li> EmCraft VF6 SOM and baseboard configs</li>
<li> Freescale SabreSD board config</li>
<li> Freescale VF65GS10 tower board config</li>
<li> Pipistrello Xilinx Spartan6 LX45 FPGA board config</li>
<li> miniSpartan6+ board config</li>
<li> Xilinx Kintex7 Development board config</li>
<li> Parallella-I board config</li>
<li> Digilent Atlys and Analog Discovery board configs</li>
<li> Numato Opsis board config</li>
<li> Xilinx Spartan 6 FPGA &#8220;Device DNA&#8221; reading support</li>
<li> Altera 10M50 FPGA (MAX10 family) target config</li>
<li> Altera EPM240 CPLD (MAXII family) target config</li>
<li> Marsohod2, Marsohod3 FPGA, Marsohod CPLD boards configs</li>
<li> Novena&#8217;s integrated FPGA board config</li>
<li> XMOS XS1-XAU8A-10&#8242;s ARM core config</li>
<li> XMOS xCORE-XA Core Module board config</li>
<li> Exynos5250 target config</li>
<li> Arndale board config</li>
<li> FM4 MB9BFxxx family configs</li>
<li> Spansion SK-FM4-U120-9B560 board config</li>
<li> Diolan LPC4357-DB1 board config</li>
<li> ST STM32F469 discovery board config</li>
<li> ST STM32F7-DISCO, STM327[4|5]6G-EVAL boards configs</li>
<li> ST STM32L4 discovery, NUCLEO L476RG, STM32F429I-DISC1 boards configs</li>
<li> Atheros AR2313, AR2315 targets config</li>
<li> Netgear WP102 board config</li>
<li> La Fonera FON2200 board config</li>
<li> Linksys WAG200G board config</li>
<li> LPC-Link2 board config</li>
<li> NXP LPC4370 target config</li>
<li> Atmel SAMV, SAMS, SAME target configs</li>
<li> Atmel SAM E70 Xplained, SAM V71 Xplained Ultra boards configs</li>
<li> Nordic nRF52 target config</li>
<li> Nordic nRF51-DK, nRF52-DK boards configs</li>
<li> Infineon XMC4700 Relax Kit, XMC4800 Relax EtherCAT Kit, XMC4300 Relax EtherCAT Kit boards configs</li>
<li> Renesas S7G2 target config</li>
<li> Renesas DK-S7G2 board config</li>
<li> Altera EP3C10 FPGA (Cyclone III family) target config</li>
<li> TI MSP432P4xx target config</li>
<li> Cypress PSoC 5LP target config</li>
<li> Analog Devices ADSP-SC58x target config (Cortex-A5 core only)</li>
</ul>
<p>
Server Layer:</p>
<ul>
<li> tcl_trace command for async target trace output via Tcl RPC</li>
</ul>
<p>
Build and Release:</p>
<ul>
<li> Various fixes thanks to <a href="http://coccinellery.org">http://coccinellery.org</a></li>
<li> libftdi is now autodetected with pkgconfig</li>
<li> Releases should now support reproducible builds</li>
<li> Conversion to non-recursive make, requires automake >= 1.14</li>
<li> Udev rules modified to add uaccess tag and moved to 60-openocd.rules</li>
<li> Support searching for scripts relative to the openocd binary for all major architectures</li>
</ul>
<p>
This release also contains a number of other important functional and cosmetic bugfixes. For more details about what has changed since the last release, see the <a href="http://sourceforge.net/p/openocd/code/ci/v0.10.0/log/?path=">git repository history</a>.
</p>
<p>
Good luck, and happy hacking!</p>

				<h2><a href="http://openocd.org/2015/05/openocd-0-9-0-release/" rel="bookmark" title="Permanent Link to OpenOCD 0.9.0 release">OpenOCD 0.9.0 release</a></h2>
				<small>May 18th, 2015 <!-- by Paul Fertser --></small>

				<div class="entry">
					<p>I&#8217;m happy to announce the release of OpenOCD version 0.9.0, finally!</p>
<p>It tooks us a bit more than a year but the list of changes isn&#8217;t a small one either. With 439 commits from 94 contributors, it&#8217;s an impressive example of how the community can drive the project forward. As an active OpenOCD user, I really appreciate all your patches, bugreports, discussions and friendly chat on IRC, thank you folks! Special mention goes to those who tirelessly review the patches and keep the project afloat as a whole: the core maintainers team.</p>
<p>We&#8217;ve also seen participation from the big semiconductor manufacturers, both nice and odd. Not everybody understands why it&#8217;s important to provide the customers with free software cross-platform tools, so please, even if you&#8217;re just a customer, you still have a voice, send your vendor an email explaining why it would be important for you to have support for their ICs in upstream OpenOCD (or if it&#8217;s already there, how happy you&#8217;re about it). The more examples they get, the more likely they&#8217;ll listen and learn; and I hope those who have already tried collaborating with the OpenOCD project enjoyed it as much as we did.</p>
<p>The release tarballs are available from the usual SF location: <a href="https://sourceforge.net/projects/openocd/files/openocd/0.9.0/">https://sourceforge.net/projects/openocd/files/openocd/0.9.0/</a> and the documentation can be easily reached from <a href="http://openocd.org/documentation/">http://openocd.org/documentation/</a> .</p>
<p>Here follows a copy of the release NEWS for your convenience.</p>
<p>Enjoy, and happy hacking!</p>
<p>JTAG Layer:</p>
<ul>
<li>SWD support with FTDI, Versaloon, J-Link, sysfsgpio</li>
<li> CMSIS-DAP massive speed and stability improvements</li>
<li> Versaloon driver ported to libusb-1.0</li>
<li> STLink can reestablish communication with a target that was disconnected or rebooted</li>
<li> STLink FAULT and WAIT SWD handling improved</li>
<li> New hla_serial command to distinguish between several HLA adapters attached to a single machine</li>
<li> Serial number support for CMSIS-DAP and J-Link adapters</li>
<li> Support for more J-Link adapters</li>
<li> TAP autoprobing improvements</li>
<li> Big speedup for SVF playback with USB Blaster</li>
</ul>
<p>Boundary Scan:</p>
<p>Target Layer:</p>
<ul>
<li> Stability improvements for targets that get disconnected or rebooted during a debug session</li>
<li> MIPS speed and reliability improvements</li>
<li> MIPS 1.5/2.0 fixes</li>
<li> ARMv7-R improvements</li>
<li> Cortex-A improvements, A7, A15 MPCores support</li>
<li> FPU support for ARMv7-M (Cortex-M4F)</li>
<li> TPIU/ITM support (including SWO/SWV tracing), can be captured with external tools or STLink</li>
<li> JTAG Serial Port (Advanced Debug System softcore) support</li>
<li> Profiling support for OpenRISC</li>
<li> ChibiOS/RT 3.0 support (with and without FPU)</li>
<li> FreeRTOS current versions support</li>
<li> Freescale MQX RTOS support</li>
<li> GDB target description support for MIPS</li>
<li> The last created target is auto-selected as the current</li>
</ul>
<p>Flash Layer:</p>
<ul>
<li> nRF51 async loader to improve flashing performance and stability</li>
<li> Cypress PSoC 41xx/42xx and CCG1 families flash driver</li>
<li> Silabs SiM3 family flash driver</li>
<li> Marvell Wireless Microcontroller SPI flash driver</li>
<li> Kinetis mass erase (part unsecuring) implemented</li>
<li> lpcspifi stability fixes</li>
<li> STM32 family sync with reference manuals, L0 support, bugfixes</li>
<li> LPC2000 driver automatically determines part and flash size</li>
<li> NXP LPC11(x)xx, LPC13xx, LPC15xx, LPC8xx, LPC5410x, LPC407x support</li>
<li> Atmel SAMD, SAMR, SAML21 devices support</li>
<li> Atmel SAM4E16 support</li>
<li> ZeroGecko family support</li>
<li> TI Tiva C Blizzard and Snowflake families support</li>
<li> Nuvoton NuMicro M051 support</li>
<li> EZR32 support in EFM32 driver </li>
</ul>
<p>Board, Target, and Interface Configuration Scripts:</p>
<ul>
<li> Normal target configs can work with HLA (STLink, ICDI) adapters</li>
<li> STM32 discovery and Nucleo boards configs</li>

<li> nRF51822-mKIT board config</li>
<li> RC Module К1879ХБ1Я target config</li>
<li> TI TMDX570LS20SUSB board config</li>
<li> TI TMS570 USB Kit board config</li>
<li> TI CC2538, CC26xx target configs</li>
<li> TI AM437x major config improvements, DDR support</li>
<li> TI AM437X IDK board config</li>
<li> TI SimpleLink Wi-Fi CC3200 LaunchPad configs</li>
<li> Silicon Labs EM357, EM358 target configs</li>
<li> Infineon XMC1000, XMC4000 family targets and boards configs</li>

<li> TP-LINK TL-MR3020 board config</li>
<li> Alphascale asm9260t target and eval kit configs</li>
<li> Olimex SAM7-LA2 (AT91SAM7A2) board config</li>

<li> Spansion FM4 target and SK-FM4-176L-S6E2CC board configs</li>

			<div class="post-340 post type-post status-publish format-standard hentry category-news" id="post-340">
				<h2><a href="http://openocd.org/2014/04/openocd-0-8-0-release/" rel="bookmark" title="Permanent Link to OpenOCD 0.8.0 release">OpenOCD 0.8.0 release</a></h2>
				<small>April 27th, 2014 <!-- by Paul Fertser --></small>

				<div class="entry">
					<p>Today I&#8217;m happy to announce the availability of OpenOCD 0.8.0 release.</p>
<p>It&#8217;s been almost a year since v0.7.0, and the amount of changes is impressive. The project gained 458 commits from 78 contributors all around the world; added to that countless hours that were spent on reviewing, discussing, testing and bug-reporting. I&#8217;d like to take this opportunity to say &#8220;thank you&#8221; to everybody involved, your participation is sincerely appreciated.</p>
<p>Of specific interest here were contributions directly from hardware manufacturers. During this development cycle OpenOCD received support from Andes Technology Corporation and Intel Corporation, and in both cases it was a fine collaboration following the best practices as established in the free software world. To help other manufacturers join the trend I propose we all start demanding out-of-the-box OpenOCD support from the vendors, educating them about Intel&#8217;s experience.</p>
<p>Please find the updated sources in the git repository and on the SF.net download area:</p>
<p><a title="http://sourceforge.net/projects/openocd/files/openocd/0.8.0/" href="http://sourceforge.net/projects/openocd/files/openocd/0.8.0/">http://sourceforge.net/projects/openocd/files/openocd/0.8.0/</a></p>
<p>Freddie Chopin has kindly provided windows binaries on his website:</p>
<p><a href="http://www.freddiechopin.info/">http://www.freddiechopin.info/</a> &gt; Download &gt; Software &gt; OpenOCD, see <a href="http://www.freddiechopin.info/en/articles/34-news/92-openocd-w-wersji-080">http://www.freddiechopin.info/en/articles/34-news/92-openocd-w-wersji-080</a> for details.</p>
<p>Good luck, and happy hacking!</p>
<p>The release highlights are listed below:</p>
<p>JTAG Layer:</p>
<ul>
<li>New CMSIS-DAP driver.</li>
<li>Andes AICE debug adapter support.</li>
<li>New OpenJTAG driver.</li>
<li>New BCM2835 (RaspberryPi) driver.</li>
<li>JTAG VPI client driver (for OpenRISC Reference Platform SoC).</li>
<li>Xilinx BSCAN_* for OpenRISC support.</li>
<li>ST-LINKv2-1 support.</li>
<li>ST-LINKv2 SWO tracing support (UART emulation).</li>
<li>JLink-OB (onboard) support.</li>
<li>Altera USB Blaster driver rewrite, initial Blaster II support.</li>
<li>ULINK driver ported to libusb-1.0, OpenULINK build fixes.</li>
<li>Support up to 64 bit IR lengths.</li>
<li>SVF playback (FPGA programming) fixes.</li>
<li>&#8220;ftdi&#8221; interface driver got extensive testing and is now recommended over the old ft2232 implementation.</li>
</ul>
<p>Boundary Scan:</p>
<p>Target Layer:</p>
<ul>
<li>New target: Andes nds32.</li>
<li>New target: OpenRISC OR1K.</li>
<li>New target: Intel Quark X10xx.</li>
<li>MIPS EJTAG 1.5/2.0 support.</li>
<li>MIPS speed improvements.</li>
<li>Cortex-M, Cortex-A (MEM-AP, APB-AP) targets working with BE hosts now.</li>
<li>XScale vector_catch support, reset fixes.</li>
<li>dsp563xx ad-hoc breakpoint/watchpoint support.</li>
<li>RTOS support for embKernel.</li>
<li>Target profiling improvements.</li>
<li>Memory access functions testbench.</li>
</ul>
<p>Flash Layer:</p>
<ul>
<li>STM32 family sync with reference manuals, other bugfixes.</li>
<li>STM32F401, STM32F07x support.</li>
<li>Atmel SAM4L, SAMG5x support.</li>
<li>Atmel AT91SAM3SD8[a,b], AT91SAM3S8[a,b,c], AT91SAM4S, AT91SAM3N0[a,b,0a,0b] support, bugfixes.</li>
<li>Atmel SAMD support.</li>
<li>Milandr 1986ВЕ* support.</li>
<li>Kinetis KL, K21 support.</li>
<li>Nuvoton NuMicro MINI5{1,2,4} support.</li>
<li>Nuvoton NUC910 series support.</li>
<li>NXP LPC43xx, LPC2000 fixes.</li>
<li>NXP LPC800, LPC810 support.</li>
<li>More ATmega parts supported.</li>
<li>Fujitsu MB9Ax family support.</li>
<li>EFM32 Wonder Gecko family support.</li>
<li>Nordic nRF51 support.</li>
</ul>
<p>Board, Target, and Interface Configuration Scripts:</p>
<ul>
<li>STM32W108xx generic target config.</li>
<li>STM32F429 discovery board config.</li>
<li>STM32 Nucleo boards configs.</li>
<li>DENX M53EVK board config.</li>
<li>Altera Cyclone V SoC, SoCkit config.</li>
<li>New TI Launchpads board configs.</li>
<li>TI am43xx devices, AM437x GP EVM, AM438x ePOS EVM board configs.</li>
<li>Marvell Armada 370 family initial support.</li>
<li>TI TMDX570LS31USB (TMS570, Cortex-R4) support scripts.</li>
<li>Freescale FRDM-KL25Z, KL46Z board configs.</li>
<li>Digilent Zedboard config.</li>
<li>Asus RT-N16, Linksys WRT54GL, BT HomeHub board configs.</li>
<li>Atmel Xplained initial support.</li>
<li>Broadcom bcm28155_ap board config.</li>
<li>TUMPA, TUMPA Lite interface configs.</li>
<li>Digilent JTAG-SMT2 interface config.</li>
<li>New RAM testing functions.</li>
<li>Easy-to-use firmware recovery helpers targeting ordinary users with common equipment.</li>
</ul>
<p>Server Layer:</p>
<ul>
<li>Auto-generation of GDB target description for ARMv7-M (XML support in GDB is mandatory for this architecture now), ARM4, nds32, OR1K, Quark.</li>
<li>GDB File-I/O Remote Protocol extension support.</li>
<li>Default GDB flashing events handlers to initialise and reset the target automatically when &#8220;load&#8221; is used.</li>
</ul>
<p>Documentation:</p>
<ul>
<li>Extensive README* changes.</li>
<li>The official User&#8217;s Guide was proofread.</li>
<li>Example cross-build script.</li>
<li>RTOS documentation improvements.</li>
<li>Tcl RPC documentation and examples added.</li>
</ul>
<p>Build and Release:</p>
<ul>
<li>*BSD, OS X, clang, ARM, windows build fixes.</li>
<li>New pkg-config support changes the way libusb (and other dependencies) are handled. Many adapter drivers are now selected automatically during the configure stage.</li>
</ul>
<p>PS: A next-release teaser: I know many people would like to see SWD support for FTDI-based adapters. It&#8217;s already working really good but not yet ready for merging, the patches are available from Gerrit and it&#8217;s likely they&#8217;ll be ready by the next release.</p>
				</div>

<li>New NXP LPC4300 support.</li>
<li>New NXP SPIFI support.</li>
<li>New Energy Micro EFM32 support.</li>
<li>New ST STM32W support.</li>

<li>Ability to override STM32 flash bank size.</li>

<p>Board, Target, and Interface Configuration Scripts:</p>
<ul>

</ul>
<p>Documentation:</p>
<ul>

<p>Build and Release:</p>
<p>For more details about what has changed since the last release, see the git repository history. With gitweb, you can browse that in various levels of detail.</p>
<p>For older NEWS, see the NEWS files associated with each release (i.e. NEWS-&lt;version&gt;).</p>
<p>For more information about contributing test reports, bug fixes, or new features and device support, please read the new Developer Manual (or the BUGS and PATCHES.txt files in the source archive).</p>
